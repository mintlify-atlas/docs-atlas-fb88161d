Shadcn Admin follows a feature-based architecture that promotes scalability and maintainability. This guide explains the project's directory structure and key organizational patterns.

## Directory Overview

The project uses a well-organized structure that separates concerns and groups related functionality:

```
src/
├── assets/           # Static assets and custom icons
├── components/       # Reusable UI components
├── context/          # React context providers
├── features/         # Feature-based modules
├── hooks/            # Custom React hooks
├── lib/              # Utility functions
├── routes/           # TanStack Router route definitions
├── stores/           # State management stores
└── styles/           # Global styles and theme
```

## Feature-Based Organization

The `features/` directory uses a modular approach where each feature is self-contained:

```bash Example: Dashboard Feature
│   ├── recent-sales.tsx
│   └── overview-chart.tsx
```

```bash Example: Users Feature
src/features/users/
├── components/
│   ├── user-form.tsx
│   └── user-details.tsx
```

```bash Example: Authentication Feature
src/features/auth/
```

<Note>
This feature-based structure makes it easy to locate related code and maintain clear boundaries between different parts of the application.
</Note>

## Key Files

### Application Entry Point

<Accordion title="src/main.tsx">
The main entry point sets up the application with all necessary providers:

```tsx src/main.tsx
import { StrictMode } from 'react'
import ReactDOM from 'react-dom/client'
import { QueryClientProvider } from '@tanstack/react-query'
import { RouterProvider, createRouter } from '@tanstack/react-router'
import { DirectionProvider } from './context/direction-provider'
import { FontProvider } from './context/font-provider'
import { ThemeProvider } from './context/theme-provider'
import { routeTree } from './routeTree.gen'
import './styles/index.css'

const router = createRouter({
  routeTree,
  context: { queryClient },
  defaultPreload: 'intent',
  defaultPreloadStaleTime: 0,
})

root.render(
  <StrictMode>
    <QueryClientProvider client={queryClient}>
      <ThemeProvider>
        <FontProvider>
          <DirectionProvider>
            <RouterProvider router={router} />
          </DirectionProvider>
        </FontProvider>
      </ThemeProvider>
    </QueryClientProvider>
  </StrictMode>
)
```
</Accordion>

<Accordion title="components/layout/">
Layout components provide the application shell:

**Example: Authenticated Layout**

```tsx components/layout/authenticated-layout.tsx
import { SidebarInset, SidebarProvider } from '@/components/ui/sidebar'
import { AppSidebar } from '@/components/layout/app-sidebar'

export function AuthenticatedLayout({ children }: AuthenticatedLayoutProps) {
  const defaultOpen = getCookie('sidebar_state') !== 'false'
  return (
    <SearchProvider>
      <LayoutProvider>
        <SidebarProvider defaultOpen={defaultOpen}>
          <AppSidebar />
          <SidebarInset>
            {children ?? <Outlet />}
          </SidebarInset>
        </SidebarProvider>
      </LayoutProvider>
    </SearchProvider>
  )
}
```
</Accordion>

## Best Practices

<Tip>
**Co-location Principle**: Keep related files close together. If a component, its styles, and its tests are only used by one feature, place them in that feature's directory.
</Tip>

<Warning>
**Avoid Deep Nesting**: Keep directory structures relatively flat (2-3 levels max) to maintain discoverability.
</Warning>

### Import Path Aliases

The project uses the `@/` alias for clean imports:

```tsx Good: Using Path Alias
import { Button } from '@/components/ui/button'
import { useTheme } from '@/context/theme-provider'
import { Dashboard } from '@/features/dashboard'
```

```tsx Avoid: Relative Imports
import { Button } from '../../../components/ui/button'
import { useTheme } from '../../context/theme-provider'
```

## Adding New Features

When creating a new feature:

<Steps>
  <Step title="Create Feature Directory">
    Add a new folder under `src/features/` with your feature name:
    ```bash
    src/features/my-feature/
    ```
  </Step>
  
  <Step title="Add Feature Structure">
    Create the standard subdirectories:
    ```bash
    src/features/my-feature/
    ├── components/    # Feature-specific components
    ├── data/          # Types, schemas, mock data
    └── index.tsx      # Main feature export
    ```
  </Step>
  
  <Step title="Create Route">
    Add a route file in `src/routes/_authenticated/my-feature/index.tsx`
  </Step>
  
  <Step title="Update Navigation">
    Add your feature to the sidebar navigation in `src/components/layout/data/sidebar-data.ts`
  </Step>
</Steps>
