## Understand the Structure

### Application Entry Point

The application starts in `src/main.tsx`:

```tsx src/main.tsx
import { RouterProvider, createRouter } from '@tanstack/react-router'
import { QueryClientProvider } from '@tanstack/react-query'
import { ThemeProvider } from './context/theme-provider'
import { routeTree } from './routeTree.gen'

const queryClient = new QueryClient()
const router = createRouter({
  routeTree,
  context: { queryClient },
})

root.render(
  <QueryClientProvider client={queryClient}>
    <ThemeProvider>
      <FontProvider>
        <DirectionProvider>
          <RouterProvider router={router} />
        </DirectionProvider>
      </FontProvider>
    </ThemeProvider>
  </QueryClientProvider>
)
```

The app is wrapped in several providers:
- **QueryClientProvider** - TanStack Query for data fetching
- **ThemeProvider** - Light/dark mode management
- **FontProvider** - Custom font configuration
- **DirectionProvider** - RTL support

### Routing Structure

Shadcn Admin uses **TanStack Router** with file-based routing. Routes are defined in `src/routes/`:

```
routes/
├── __root.tsx              # Root layout
├── _authenticated/         # Protected routes
│   ├── route.tsx          # Layout with sidebar
│   ├── index.tsx          # Dashboard (/)
│   ├── tasks/             # Tasks pages
│   ├── users/             # Users pages
│   └── settings/          # Settings pages
├── (auth)/                # Auth pages (no layout)
│   ├── sign-in.tsx
│   └── sign-up.tsx
└── (errors)/              # Error pages
    ├── 404.tsx
    └── 500.tsx
```

## Make Your First Customization

Let's customize the sidebar by adding a new navigation item.

<Steps>
  <Step title="Open the Sidebar Data File">
    Open `src/components/layout/data/sidebar-data.ts` in your editor.
  </Step>

  <Step title="Add a New Navigation Item">
    Add a new item to the `General` nav group:

    ```typescript src/components/layout/data/sidebar-data.ts
    import { FileText } from 'lucide-react' // Add this import

    export const sidebarData: SidebarData = {
      // ... existing config
      navGroups: [
        {
          title: 'General',
          items: [
            {
              title: 'Dashboard',
              url: '/',
              icon: LayoutDashboard,
            },
            {
              title: 'Tasks',
              url: '/tasks',
              icon: ListTodo,
            },
            // Add this new item
            {
              title: 'Reports',
              url: '/reports',
              icon: FileText,
            },
            // ... other items
          ],
        },
      ],
    }
    ```
  </Step>

  <Step title="Save and Check">
    Save the file and check your browser. You should see the new "Reports" item in the sidebar.

    <Tip>
      Vite's Hot Module Replacement (HMR) will automatically update the page without a full refresh.
    </Tip>
  </Step>

  <Step title="Create the Route (Optional)">
    To make the link functional, create a new route file:

    ```tsx src/routes/_authenticated/reports.tsx
    import { createFileRoute } from '@tanstack/react-router'

    export const Route = createFileRoute('/_authenticated/reports')({
      component: ReportsPage,
    })

    function ReportsPage() {
      return (
        <div className="space-y-4">
          <h1 className="text-3xl font-bold">Reports</h1>
          <p className="text-muted-foreground">
            Your reports will appear here.
          </p>
        </div>
      )
    }
    ```
  </Step>
</Steps>

## Customize the Theme

Shadcn Admin uses CSS variables for theming, making it easy to customize colors.

### Change Primary Color

Edit `src/styles/index.css` to modify theme colors:

```css src/styles/index.css
@layer base {
  :root {
    --primary: 222.2 47.4% 11.2%;        /* Default primary color */
    --primary-foreground: 210 40% 98%;   /* Text on primary */
    /* Change to a blue theme: */
    /* --primary: 221 83% 53%; */
  }
}
```

<Tip>
  Use tools like [uicolors.app](https://uicolors.app) to generate HSL color values that work well with Shadcn UI.
</Tip>

## Working with Components

Shadcn Admin includes all Shadcn UI components in `src/components/ui/`. These are highly customizable and accessible.

### Using a Component

Here's an example of using the Button and Card components:

```tsx
import { Button } from '@/components/ui/button'
import { Card, CardHeader, CardTitle, CardContent } from '@/components/ui/card'

      </CardHeader>
      <CardContent>

}
```

## RTL Support

<Note>
  Shadcn Admin includes built-in RTL (Right-to-Left) support for Arabic, Hebrew, and other RTL languages.
</Note>

Several components have been customized for RTL support:

- **Modified Components**: scroll-area, sonner, separator
- **RTL Updated Components**: alert-dialog, calendar, command, dialog, dropdown-menu, select, table, sheet, sidebar, switch

To enable RTL mode, the `DirectionProvider` in `src/context/direction-provider.tsx` handles the layout direction.

## Common Tasks

<AccordionGroup>
  <Accordion title="How do I add a new page?">
    1. Create a new file in `src/routes/_authenticated/`
    2. Define the route using `createFileRoute`
    3. Add a navigation item in `src/components/layout/data/sidebar-data.ts`
    
    Example:
    ```tsx
    // src/routes/_authenticated/my-page.tsx
    import { createFileRoute } from '@tanstack/react-router'

    export const Route = createFileRoute('/_authenticated/my-page')({
      component: MyPage,
    })

    function MyPage() {
      return <div>My new page</div>
    }
    ```
  </Accordion>

  <Accordion title="How do I change the logo?">
    The logo is configured in the TeamSwitcher component. Edit:
    - `src/components/layout/team-switcher.tsx` - Component implementation
    - `src/components/layout/data/sidebar-data.ts` - Team/logo data
  </Accordion>

  <Accordion title="How do I modify the dashboard charts?">
    Dashboard charts are in `src/features/dashboard/`. They use Recharts library. Modify the data and configuration in the respective component files.
  </Accordion>

  <Accordion title="How do I change the sidebar behavior?">
    The sidebar uses the `useSidebar` hook from `src/components/ui/sidebar.tsx`. You can customize:
    - **Collapsible mode**: Set in `src/components/layout/app-sidebar.tsx`
    - **Variant**: Changed via the config drawer or layout provider
    - **Keyboard shortcut**: Defined as `SIDEBAR_KEYBOARD_SHORTCUT` in sidebar.tsx (default: 'b')
  </Accordion>
</AccordionGroup>
