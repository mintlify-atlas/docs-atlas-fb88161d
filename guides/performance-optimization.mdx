## Overview

Shadcn Admin is built with performance in mind, using modern tools and techniques for fast load times and smooth interactions. This guide covers built-in optimizations and advanced techniques.

## Built-in Optimizations

### Automatic Code Splitting

TanStack Router automatically splits your application by route:

```ts vite.config.ts
import { tanstackRouter } from '@tanstack/router-plugin/vite'

export default defineConfig({
  plugins: [
    tanstackRouter({
      target: 'react',
      autoCodeSplitting: true, // Automatic route-based code splitting
    }),
  ],
})
```

**Benefits:**
- Each route loads only when needed
- Smaller initial bundle size
- Faster first contentful paint
- Better caching granularity

### React Fast Refresh with SWC

The project uses SWC instead of Babel for up to 20x faster builds:

```ts vite.config.ts
import react from '@vitejs/plugin-react-swc'

export default defineConfig({
  plugins: [react()],
})
```

**Advantages:**
- Lightning-fast hot module replacement
- Faster production builds
- Lower memory usage during development

## React Performance Patterns

### Memoization

The codebase uses memoization to prevent unnecessary re-renders:

```tsx src/context/theme-provider.tsx
import { useMemo } from 'react'

export function ThemeProvider({ children, defaultTheme }) {
  const [theme, _setTheme] = useState(defaultTheme)

  // Memoize expensive calculations
  const resolvedTheme = useMemo((): ResolvedTheme => {
    if (theme === 'system') {
      return window.matchMedia('(prefers-color-scheme: dark)').matches
        ? 'dark'
        : 'light'
    }
    return theme as ResolvedTheme
  }, [theme])

  // ... rest of component
}
```

### Lazy Loading Components

Lazy load heavy components that aren't immediately needed:

```tsx
import { lazy, Suspense } from 'react'

// Lazy load chart component
const RevenueChart = lazy(() => import('./components/revenue-chart'))

function Dashboard() {
  return (
    <Suspense fallback={<ChartSkeleton />}>
      <RevenueChart data={data} />
    </Suspense>
  )
}
```

### useCallback for Event Handlers

Prevent function recreation on every render:

```tsx
import { useCallback } from 'react'

function UserList() {
  const [users, setUsers] = useState([])

  // Memoize callback to prevent child re-renders
  const handleDelete = useCallback((id: string) => {
    setUsers(prev => prev.filter(u => u.id !== id))
  }, [])

  return (
    <>
      {users.map(user => (
        <UserCard key={user.id} user={user} onDelete={handleDelete} />
      ))}
    </>
  )
}
```

### React.memo for Pure Components

Memoize components that render the same output for the same props:

```tsx
import { memo } from 'react'

interface UserCardProps {
  user: User
  onDelete: (id: string) => void
}

export const UserCard = memo(function UserCard({ user, onDelete }: UserCardProps) {
  return (
    <div className="card">
      <h3>{user.name}</h3>
      <button onClick={() => onDelete(user.id)}>Delete</button>
    </div>
  )
})
```

## TanStack Query Optimizations

### Query Configuration

Optimize data fetching with proper configuration:

```tsx src/main.tsx
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 10 * 1000, // Data fresh for 10 seconds
      refetchOnWindowFocus: import.meta.env.PROD, // Only in production
      retry: (failureCount, error) => {
        // Smart retry logic
        if (failureCount >= 0 && import.meta.env.DEV) return false
        if (failureCount > 3 && import.meta.env.PROD) return false
        return !(
          error instanceof AxiosError &&
          [401, 403].includes(error.response?.status ?? 0)
        )
      },
    },
  },
})
```

### Prefetching Data

Prefetch data before users navigate:

```tsx
import { useQueryClient } from '@tanstack/react-query'

function UserList() {
  const queryClient = useQueryClient()

  const handleUserHover = (userId: string) => {
    // Prefetch user details on hover
    queryClient.prefetchQuery({
      queryKey: ['users', userId],
      queryFn: () => fetchUser(userId),
    })
  }

  return (
    <div>
      {users.map(user => (
        <div 
          key={user.id} 
          onMouseEnter={() => handleUserHover(user.id)}
        >
          {user.name}
        </div>
      ))}
    </div>
  )
}
```

### Pagination and Infinite Queries

Use pagination for large datasets:

```tsx
import { useInfiniteQuery } from '@tanstack/react-query'

function InfiniteUserList() {
  const {
    data,
    fetchNextPage,
    hasNextPage,
    isFetchingNextPage,
  } = useInfiniteQuery({
    queryKey: ['users'],
    queryFn: ({ pageParam = 1 }) => fetchUsers({ page: pageParam }),
    getNextPageParam: (lastPage, pages) => lastPage.nextPage,
    initialPageParam: 1,
  })

  return (
    <div>
      {data?.pages.map(page =>
        page.users.map(user => <UserCard key={user.id} user={user} />)
      )}
      {hasNextPage && (
        <button onClick={() => fetchNextPage()} disabled={isFetchingNextPage}>
          Load More
        </button>
      )}
    </div>
  )
}
```

### Tree Shaking

Ensure imports are tree-shakeable:

```tsx
// ✅ Good - tree-shakeable
import { Button } from '@/components/ui/button'
import { Card } from '@/components/ui/card'

// ❌ Bad - imports everything
import * as UI from '@/components/ui'
```

### Dynamic Imports

Split heavy dependencies:

```tsx
// Only load chart library when needed
const loadChartLibrary = async () => {
  const { Chart } = await import('recharts')
  return Chart
}

function ChartComponent() {
  const [Chart, setChart] = useState(null)

  useEffect(() => {
    loadChartLibrary().then(setChart)
  }, [])

  if (!Chart) return <Skeleton />
  return <Chart data={data} />
}
```

### Resource Hints

Add resource hints to `index.html`:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Preconnect to external domains -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="dns-prefetch" href="https://api.example.com">
    
    <!-- Preload critical assets -->
    <link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>
  </head>
  <body>
    <div id="root"></div>
  </body>
</html>
```

Enable gzip/brotli compression on your server:

```nginx nginx.conf

brotli on;

```

## Runtime Performance

### Virtual Scrolling

For large lists, use virtual scrolling:

```bash
pnpm add @tanstack/react-virtual
```

```tsx
import { useVirtualizer } from '@tanstack/react-virtual'

function VirtualList({ items }) {
  const parentRef = useRef(null)

  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,
  })

  return (
    <div ref={parentRef} style={{ height: '400px', overflow: 'auto' }}>
      <div style={{ height: `${virtualizer.getTotalSize()}px` }}>
        {virtualizer.getVirtualItems().map(virtualItem => (
          <div
            key={virtualItem.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              transform: `translateY(${virtualItem.start}px)`,
            }}
          >
            {items[virtualItem.index].name}
          </div>
        ))}
      </div>
    </div>
  )
}
```

### Debouncing and Throttling

Optimize expensive operations:

```tsx
import { useMemo } from 'react'

function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value)

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value)
    }, delay)

    return () => clearTimeout(handler)
  }, [value, delay])

  return debouncedValue
}

// Usage in search
function SearchUsers() {
  const [search, setSearch] = useState('')
  const debouncedSearch = useDebounce(search, 300)

  const { data } = useQuery({
    queryKey: ['users', debouncedSearch],
    queryFn: () => fetchUsers({ search: debouncedSearch }),
  })

  return <input value={search} onChange={e => setSearch(e.target.value)} />
}
```
